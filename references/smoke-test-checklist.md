# 冒烟测试清单

## 目的

在 Step 2.5 阶段快速验证核心功能，避免明显缺陷进入 Step 3 审查。

## 核心功能路径

- [ ] **主要功能正常执行**（happy path）
  - 核心业务逻辑可以完整执行
  - 无运行时错误或异常
- [ ] **输出格式符合预期**
  - 返回值类型正确
  - 数据结构完整

## 边界条件

- [ ] **空输入处理**
  - 空字符串、空列表、空字典
  - 函数不崩溃，返回合理默认值或错误提示
- [ ] **极值输入处理**
  - 最大值、最小值、负数、零
  - 边界值不导致溢出或异常
- [ ] **异常输入处理**
  - 非法格式（类型错误、格式错误）
  - 优雅降级或抛出明确错误信息

## 依赖检查

- [ ] **外部服务可访问**
  - API 端点可连接
  - 认证凭证有效
- [ ] **配置文件存在且合法**
  - 必要的配置文件存在
  - 配置项格式正确、值合法
- [ ] **必要的环境变量已设置**
  - 检查 required env vars
  - 值不为空且格式正确

## 性能基准（可选，L2/L3 推荐）

- [ ] **响应时间 < 基准 +20%**
  - 关键函数执行时间不超过基准 1.2 倍
- [ ] **内存使用 < 基准 +30%**
  - 峰值内存不超过基准 1.3 倍

## 冒烟失败报告格式

当冒烟测试失败时，coding 必须输出结构化报告：

```json
{
  "status": "SMOKE_FAILED",
  "failed_items": [
    {
      "category": "边界条件",
      "item": "空输入处理",
      "error": "TypeError: 'NoneType' object is not iterable",
      "location": "src/utils.py:42",
      "suggestion": "添加空值检查：if data is None: return []"
    }
  ],
  "passed_items": ["核心功能路径", "依赖检查"],
  "logs": "完整错误日志..."
}
```

## 执行规范

1. coding 在 Step 2 完成后，自动执行此清单
2. 所有项通过 → 创建 S1 快照，进入 Step 3
3. 任何项失败：
   - **语法/编译错误**：coding 自修复（max 2 次）
   - **逻辑/功能错误**：输出失败报告，进入 Step 4
4. 2 次自修复仍失败 → 输出失败报告，进入 Step 4

## 注意事项

- 冒烟测试不是完整测试，只验证核心路径
- 目标是快速发现明显缺陷，不是 100% 覆盖
- L1 任务可简化清单（只检查核心功能路径）
- L2/L3 任务建议执行完整清单
